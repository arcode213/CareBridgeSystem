The Referral Code is the Single Source of Truth

Every patient, every report, every bill, every payment, and every audit trail in CareBridge is anchored to one unique REF-XXXXX code. No duplicate records. No disconnected data. One code links the entire patient journey — from consultant referral to hospital discharge to lab results to financial settlement.

§2 — Centralized Referral Code Logic

The universal patient identity anchoring every record in the platform

MODULE 02

Every referral generated by a Consultant automatically creates a unique Referral Code. This becomes the patient's centralized identity across the entire platform. All medical history, reports, hospital activity, diagnostics, billing, and financial tracking remain connected to this single code.

⚠️

Access Control Rule: Patient details are NOT visible to any hospital or laboratory until they enter the unique Referral Code. Only after code entry will all linked patient data become accessible.

§7 — Locked Record Management

Immutable record protection and controlled unlock workflow

MODULE 07

Once patient records, referrals, or discharge summaries are finalized, the system automatically locks sensitive records. Changes require Super Admin authorization. Only Super Admin can unlock, edit, and relock records — with full audit logging at every step.

Record Lock & Unlock Process

⚙️ Record Lifecycle: Finalization → Lock → Unlock Request → Edit → Relock

1
Record Finalization

Referral accepted, discharge summary submitted, or lab report validated → record automatically transitions to "Finalized" state and triggers lock.

2
System Auto-Lock Applied

System locks the record against further edits. All downstream modules see the record as read-only. Lock timestamp and triggering user ID recorded in audit trail.

3
User Submits Unlock Request

Any module user (Hospital, Consultant, Lab) who needs a correction submits an unlock request via the platform with a stated reason. Request goes directly to Super Admin queue.

4
Super Admin Reviews & Unlocks

Super Admin reviews the unlock reason, validates legitimacy, and unlocks the record. Unlock action is logged with Admin ID, timestamp, and justification. User notified immediately.

5
Edit Window Open

Authorized user makes the required corrections within a controlled edit window. All changes are tracked (diff-level audit log) — original values preserved alongside new values.

6
Super Admin Relocks Record

After edit confirmation, Super Admin relocks the record. New lock timestamp applied. Full unlock-edit-relock cycle permanently recorded in the compliance audit trail.

📜

The lock system ensures data integrity and regulatory compliance. No record can be silently altered after finalization — every change requires a documented, admin-authorized unlock request.

§8 — Financial Workflow

Discharge billing, share calculation, and platform tracking

MODULE 08

When a patient is discharged, the Hospital uploads the final bill inside the platform. The uploaded bill is reviewed by Super Admin. After verification, the system automatically calculates the Consultant share, Hospital payable amount, and platform financial tracking — all linked to the Referral Code.

🏨

1. Hospital Uploads Final Bill

Hospital submits itemized discharge bill inside the platform, tagged to the patient's Referral Code. Bill includes procedure costs, room charges, investigations, pharmacy, and any additional fees.
