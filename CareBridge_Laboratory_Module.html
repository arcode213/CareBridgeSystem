CareBridge Laboratory Module — Operational Bridge Between Clinician & Diagnostics

The Laboratory module connects diagnostic facilities to the CareBridge Health network through a single Referral Code architecture. Labs access complete patient history, receive investigation orders, process samples, and deliver validated results — all without re-entering patient data or creating duplicate records. The system is designed to be fast, technician-friendly, and operationally precise.

🏥

§1 — Product Vision & Overview

Purpose, scope, and operational goals of the Laboratory module

MODULE 01

The CareBridge Laboratory Dashboard is built for diagnostic laboratories, pathology labs, imaging centers, collection facilities, and specialized testing environments. It bridges the gap between clinical referrals and diagnostic delivery — ensuring every investigation ordered is tracked, processed, and reported within committed turnaround times.

🎯

The laboratory dashboard is not a standalone system. It is a connected operational node within the CareBridge ecosystem — every sample, report, and alert it generates is automatically linked to the patient's Referral Code and reflected on the Consultant and Hospital dashboards in real time.

Module Scope

§2 — Referral Code Access Control

Patient data is completely gated behind the unique Referral Code

MODULE 02

Patient details are not visible to any laboratory until the unique Referral Code is entered. Only after entering the code will all linked patient details — added by the consultant — become accessible to lab staff.

⚠️

Access Control Rule: No patient name, diagnosis, or clinical history is searchable or visible in the lab module without prior code entry. This protects patient privacy and ensures only authorized, referred labs access clinical data.

§3 — Laboratory Notification System

Multi-channel real-time alerts for incoming referrals and investigation orders

MODULE 03

When a consultant sends an investigation referral to a laboratory, the lab receives three simultaneous notification channels to ensure no order is missed — especially for emergency and STAT cases.

💬

WhatsApp Notification

Automatic WhatsApp message sent to the lab's registered number. Contains patient Referral Code, investigation type, urgency level, and consultant name. Delivered instantly on referral creation.

🖥️

In-App Popup Alert

Real-time popup on the Lab Dashboard showing investigation order details, urgency flag, TAT expectation, and quick-action buttons (Accept / Acknowledge / Route to Section).

🔊

Notification Tone

Audible alert plays on the lab dashboard for new incoming orders — a distinct louder tone activates for STAT / Emergency investigations to prompt immediate action.

§4 — Sample Tracking Workflow

Complete 6-stage sample journey from order receipt to result delivery

MODULE 04

The workflow guides lab staff through every stage of diagnostic operations. Each stage updates the patient's centralized record in real time. Emergency STAT samples skip standard queues and are automatically elevated to priority processing.

Full Sample Journey
📥

Stage 1 — Order / Referral Received

Investigation request arrives via CareBridge referral, walk-in with Referral Code, or hospital order. System displays order details: tests required, urgency level, consultant name, and patient summary. STAT orders are automatically flagged with a red priority banner and notification tone.

🔑

Stage 2 — Referral Code Entry & Patient Registration

Lab staff enter the Referral Code. Full patient profile, consultant diagnosis, investigation instructions, and previous reports are loaded instantly — no re-entry of patient demographics required. New walk-in patients without a code are registered fresh and a new Referral Code is generated.

code_gateno_duplicatehistory_loaded
🩸

Stage 3 — Sample Collection

Sample collected from patient by phlebotomist or collection staff. Barcode label generated and printed — linked to the patient's Referral Code and specific test order. Collection timestamp recorded. TAT countdown activates from this exact moment. Multiple samples for the same patient grouped under one Referral Code.

barcode_generatedtat_activatedcollector_id_logged
⚙️

Stage 4 — Sample Processing

Sample routed to the relevant laboratory section (biochemistry, haematology, microbiology, radiology, etc.) based on the investigation type. Technician updates sample status on receipt. STAT samples automatically bypass standard queue and go to immediate processing. TAT timer visible on every technician's workbench view.

section_routingstat_bypass_queuetat_visible
🔍

Stage 5 — Quality Review & Validation

Senior technician or pathologist validates the results. Abnormal or critical values trigger an immediate escalation alert to the requesting consultant — push notification sent instantly. QC-failed samples automatically trigger a resample request with the reason logged. Validation is a two-person sign-off for critical investigations.

Validated report uploaded digitally against the Referral Code. PDF report auto-generated and attached to the patient's centralized record. Consultant receives instant push notification with report summary and direct access link. Hospital staff see report status update on the admitted patient's panel. TAT completion time logged for performance tracking.

§6 — Sample Status States

Real-time sample tracking states and required actions

MODULE 06

Every sample has a clearly defined status visible to lab staff, the requesting consultant, and the hospital dashboard. Status transitions trigger automatic notifications to relevant parties.

🔵 Order Received

Investigation request logged. Awaiting code entry and sample collection. TAT pending.

🟡 Awaiting Collection

Code entered, patient registered. Phlebotomist dispatched. Sample not yet collected.

🟠 Collected

Sample collected and barcoded. TAT timer active. Awaiting transport to section.

🔵 In Processing

Sample received in section. Analysis underway. TAT countdown visible to all.

🟣 Awaiting Validation

Analysis complete. Awaiting senior technician or pathologist sign-off.

🟢 Completed

Report validated, uploaded, and delivered. Consultant notified. TAT record closed.

🔴 Critical Value

Panic value detected. Consultant immediately notified via push + escalation log. Urgent action required.

⚪ QC Failed / Resample

Quality control failed. New sample required. Reason logged. Consultant and collection team notified.

⚡ STAT Priority

Emergency order. Bypasses all standard queues. All staff alerted. TAT measured in minutes, not hours.

🔄 Repeat / Add-On

Consultant added a new test to existing sample. Registered under same Ref Code. No re-collection needed.

§8b — Consultant: Create Lab Referral for Tests

How a consultant initiates a laboratory investigation order directly from their CareBridge dashboard

MODULE 08b

Consultants have a dedicated "Create Lab Referral" action within their CareBridge dashboard. This allows them to formally order one or more laboratory investigations for a patient and route the request — along with the patient's Referral Code — directly to a chosen laboratory registered on the CareBridge network.

🎯

Key Principle: When a consultant creates a lab referral, the laboratory receives the order instantly via all three notification channels (WhatsApp, in-app popup, and audio tone) and the patient's Referral Code is transmitted automatically — no phone calls or manual communication required.

Step-by-Step: Consultant Creates a Lab Referral
👤

Step 1 — Open Patient Record

The consultant navigates to the patient's profile from their active patient list or searches by Referral Code. The full patient history, diagnosis notes, and any previously uploaded lab reports are visible before placing the new order.

patient_profileref_code_loadedhistory_visible
➕

Step 2 — Click "Create Lab Referral"

A prominent "+ Create Lab Referral" button is present on every patient's profile panel. Clicking it opens the lab referral form as a modal overlay — no page navigation required. The patient's Referral Code is pre-filled and locked.

modal_overlayref_code_prefilledno_navigation
🔬

Step 3 — Select Tests / Investigations

Consultant selects one or more tests from the CareBridge investigation catalogue — organized by department (Biochemistry, Haematology, Microbiology, Radiology, etc.). Tests can be added individually or as pre-defined panels (e.g., Full Blood Count Panel, Liver Function Panel). A free-text field allows custom investigation instructions.

test_cataloguepanel_presetscustom_instructionsmulti_select
🏥

Step 4 — Choose Laboratory

Consultant selects the target laboratory from a searchable directory of CareBridge-registered labs. Labs can be filtered by specialty (e.g., Microbiology, Radiology), location, or facility name. If the patient has visited a lab before, it appears as a suggested lab at the top of the list. Multiple labs can be selected for multi-section orders (e.g., blood work at Lab A and imaging at Lab B).

lab_directoryfilter_by_specialtysuggested_labmulti_lab_support
⚡

Step 5 — Set Urgency Level

The consultant assigns a priority level to the order. This controls the notification type sent to the lab and determines how the order appears in the lab's queue.

urgency_flagstat_escalationtat_expectation_set
📝

Step 6 — Add Clinical Notes (Optional)

Consultants can attach a clinical context note to the referral — provisional diagnosis, relevant history, medication that may affect results, or specific instructions to the lab technician (e.g., "Fasting sample required", "Patient on anticoagulants"). These notes are visible to lab staff upon code entry but are read-only — labs cannot edit clinical notes.

clinical_contextread_only_for_labfasting_flag
📤

Step 7 — Submit Referral

Consultant reviews the order summary and clicks "Send Referral". The system creates an investigation order record, attaches it to the patient's Referral Code, and dispatches the order to the selected laboratory instantly. The referral is timestamped and logged in the consultant's audit trail.

order_createdlab_notifiedaudit_loggedtimestamp_recorded
📡

Step 8 — Live Tracking on Consultant Dashboard

After submission, the referral appears on the consultant's patient panel under "Pending Lab Orders". The consultant can see real-time status updates (Order Received → Collected → In Processing → Completed) without any action from the lab needing to be manually communicated. When the report is ready, the consultant receives an automatic push notification.

§9 — UX Design Principles

Human-centered design for high-volume, fast-paced diagnostic workflows

MODULE 09

The laboratory dashboard must feel operationally smooth, technician-friendly, and optimized for high-volume workflows. Lab technicians work under time pressure, with multiple samples in progress simultaneously. The interface must minimize cognitive load and physical interaction time.

⌨️

Minimal Typing

Pre-populated fields from the Referral Code. Barcode scan registers samples. Quick-select for test panels instead of manual entry.

⏱️

TAT Always Visible

Turnaround time timer displayed prominently on every sample card. Color changes from green → amber → red as deadline approaches.

🚨

STAT Visual Priority

Emergency orders use distinct red banner, pulsing icon, and audible tone. Never lost in a standard work queue.

📱

Mobile-Accessible

Collection staff and on-call technicians can update sample status from mobile — large buttons, simple status updates.

🔍

Instant Search

Referral Code search returns full patient record in under 1 second. No multi-field search needed — one code finds everything.

📊

Workbench View

Technician sees only their section's pending samples. Reduces noise, increases focus on relevant work queue.

🎯

One-Click Upload

Report upload requires minimal steps — select file, confirm patient match, one-click submit. PDF generated automatically.

📡

Real-Time Dashboard

No page refresh needed. New orders appear live. Status updates propagate instantly across all connected user sessions.
