1. Project Overview & Architecture

Platform Vision: CareBridge Health is a comprehensive Healthcare Referral Management SaaS platform designed to bridge the gap between doctors (consultants), hospitals (IPD), laboratories, and patients in Pakistan. The platform enables seamless patient referrals, real-time bed management, commission tracking, and emergency escalation.

1.4 Key Business Rules (From Original Portal Document)

Critical Business Logic:
  • System-generated first password for all users (Doctor, Hospital, Admin)
  • One doctor can create multiple portal IDs with different usernames/passwords
  • Patient records are end-to-end encrypted between Doctor, IPD, and Admin
  • Patient Guardian ID and CNIC required for tracking
  • Unique serial ID generated for every patient
  • Real-time updates shared between Doctor portal and IPD portal immediately upon reservation
  • Admin CANNOT interfere in treatment activity - only monitoring and authorization
  • Commission records display ONLY in admin portal
  • Hospital pays doctor through bank account; platform commission recorded separately
  • Platform bank account API connected in Admin portal

6.3 Emergency Escalation Rules (Auto-Triggered)

RULE 1: IF no hospital response in 3 minutes → Expand to Top 10 hospitals RULE 2: IF Emergency priority detected → Auto-assign nearest facility with capacity RULE 3: IF ICU bed unavailable → Search HDU / NICU / PICU alternatives RULE 4: IF all beds full in network → Escalate to network admin + SMS alert RULE 5: IF SLA timer expires → Auto-escalate + mark as critical

17. Security Requirements

17.1 Authentication & Authorization

Requirement Implementation Priority
JWT Token Expiry Access token: 15 min, Refresh token: 7 days Critical
Password Policy Min 8 chars, uppercase, lowercase, number, special char Critical
OTP Verification SMS OTP for critical actions (withdrawal, password reset) Critical
Role-Based Access Granular permissions per role, middleware enforcement Critical
Session Management Max 3 concurrent sessions, force logout capability High

17.2 Data Security

Encryption Requirements

  • End-to-end encryption for patient records
  • AES-256 for data at rest
  • TLS 1.3 for data in transit
  • Encrypted database backups
  • Secure key management (AWS KMS)

Data Protection

  • Patient data anonymization for analytics
  • CNIC masking in logs (show last 4 only)
  • Automatic logout after 30 min inactivity
  • IP whitelisting for admin access
  • Rate limiting on all APIs

17.3 Compliance Requirements

Standard Requirement Implementation
Patient Data Privacy End-to-end encryption, no unauthorized access Encrypted fields, access logs
PMDC Verification Validate all doctor licenses Integration with PMDC database
SHCC Verification Validate all hospital licenses Document upload + manual verification
Audit Trail Log all data access and modifications Immutable audit log collection
Data Retention Configurable retention policies Auto-archive after retention period
From Portal Document: "The patient and recommendation records would be end-to-end encrypted between Doctor, IPD and Admin and the details shall not be breach." This is a LEGAL REQUIREMENT, not optional.
